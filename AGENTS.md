# Visual & Copy Rules

## Content Style & Vocabulary
- Stick to plain, functional verbs (e.g., *Build*, *Ship*, *Deploy*, *Commit*, *Review*, *Iterate*, *Launch*) for button labels, section headings, and meta details platform-wide.
- Avoid flowery SaaS marketing adjectives or hype-focused copy.

## Imagery & Illustrations
- Never use stock education illustrations, abstract geometric vectors, or classroom photography anywhere on the platform.
- UI components should rely entirely on flat cards, solid borders, clean typography, functional icons, and live code log elements for visual appeal.
